## Context

The article template already uses Nuxt Content and Nuxt UI's table-of-contents component. Desktop pages reserve a right sidebar for the table of contents, while mobile renders an inline table of contents above the article body. Global article Markdown styling lives in `app/assets/css/main.css` under `.article-prose`.

## Goals / Non-Goals

**Goals:**

- Preserve the current quiet, text-focused visual style.
- Reduce mobile article preamble height by making the inline table of contents collapsed by default.
- Keep code blocks and collapsed long-code examples visually bounded within the article column.
- Verify the Markdown style guide article on desktop and mobile with Playwright MCP.

**Non-Goals:**

- Redesign the site navigation, article cards, or homepage.
- Add new Nuxt modules or dependencies.
- Change content schemas, routes, or article Markdown files.

## Decisions

- Use the existing `UContentToc` behavior instead of a custom mobile table-of-contents component. This keeps the change small and aligned with the current Nuxt UI usage.
- Adjust prose CSS globally under `.article-prose` because code blocks are generated by Nuxt Content and can appear in any article.
- Prefer containment and spacing around code blocks over changing Markdown content. This handles current and future posts consistently.

## Risks / Trade-offs

- Collapsing the mobile table of contents hides section links until tapped. Mitigation: the visible "目录" trigger remains above the article body.
- Global prose CSS can affect all article code examples. Mitigation: keep selectors scoped to `.article-prose` and code-related Nuxt Content structures only.
